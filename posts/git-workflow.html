Git Workflow

Andy Vuong | Copyright 2015

This document is a quick guide to getting started with git and github project collaboration by introducing simple but quick workflows you can use to easily work with others in a central repository. Note: This is not a complete guide to using Git. Have any further questions? Check out some online Git resources or Google it!

WORKFLOWS

  • 1. Collaborate on an existing project
  • 2. Collaborate using branches

Collaborate on an existing project

  • 1. Clone the project to your working directory. You now have a copy of the repo and can push changes to it that will be available to all your team members.

    Use: git clone [repo_location.git]
    Example: git clone https://github.com/user/project.git

  • 2. Add files to be commited to the main repository.

    Use: git add [filename]
    Example: git add code.c

  • 3. Use git commit to commit your code. Using git commit . - will commit all added files.

    Use: git commit [filename]
    Example git commit .

  • 4. Publish changes to the central repository so otehrs may access the changes.

    Use: git push origin master
    Note: origin is a shortcut for your clone of the repo and master is a shortcut to the central repo.

  • 5. If someone else has published their changes to the central repo, your local copy is now outdated so you'll be unable to push your own changes. You'll need to use git pull to grab the updates from the central repo and then integrate changes you made before publishing.

    Use: git pull --rebase origin master
