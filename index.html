  • Hubble [Deployment] [Github] [Blog Post]

    A Twitter data visualization web app. See the most recent tweets based on the current Twitter trends from across the globe and where those tweets originated from.

    [Read more...]

    This is a data visualization web application built using Spring Boot, D3.js, planetary.js, and MongoDB. The Spring backend consists of a RESTful API utilized by the front-end and D3.js to render visualizations on an HTML5 canvas. The Spring backend periodically runs tasks to refresh a MongoDB datastore with data from Twitter's streaming API. There are various features on the web application including the ability to see where tweets originate from on a globe, view tweets specifically from within the U.S., and view the most recent tweets from top news websites as followed by the acommpanying Twitter handle for the app: @hubble_watches.

  • Safe Route Recommender [Deployment] [Github] [Blog Post]

    A web application that computes a statistically safe path to walk from one destination to another on the University of Illinois's campus. The route is based on collected crime data.

    [Read more...]

    This is a route mapping web application built using NodeJs, Express, and the Google Maps API. Crime data for the Urbana-Champaign (Il) area is collected from a few different sources and conformed into a data object that it is stored in MongoDB. When a user inputs a origin and destination to the map, the data pulled from our database is pulled into an in-memory graph on the backend. The graph vertices represent street intersections and edges represent a path from one intersection to another. Each edge has a weight representing its danger based on the data. The path of least danger is returned to the user. See the readme on github to learn about how danger is calculated and more technical details such as how locations that are not on an intersection are represented in the graph.
